OCR Segmentation Micro-Service


GOAL

Extract the individual line images from the binarized image(s).

USAGE

1. Call OCR segmentation micro-service with url: "http://10.5.146.92:8002/segmentationapi".

2. Input:

1) The path of the binarized image(s), and uploaded in form style. Usually the images are the output of binarization service with extension ".bin.png".
2) Optional. Specified parameter valuses which are listed in the following part, and send them in form style.

3. Output:

A zip file containing equal number of folders to the number of input images. Each folder contains the extracted line images, and usually they are named as {imagename}_01????.bin.png

4. E.g. curl -F "image=@path_to_image1" -F "image=@path_to_image2" -F "threshold=0.5" -o name_of_output_file http://10.5.146.92:8002/segmentationapi

PARAMETERS (#12)

Parameter Type Default value Comment
maxlines Integer 300 maximum # lines permitted
scale Float 0.0 the basic scale of the document (roughly, xheight) 0=automatic
hscale Float 1.0 non-standard scaling of horizontal parameters
vscale Float 1.0 non-standard scaling of vertical parameters") # line parameters
threshold Float 0.2 baseline threshold
noise Integer 8 noise threshold for removing small components from lines
usegause Boolean True use gaussian instead of uniform
maxseps Integer 2 maximum # black column separators
sepwiden Integer 10 widen black separators (to account for warping)
maxcolseps Integer 3 maximum # whitespace column separators
csminheight Float 10.0 minimum column height (units=scale)
parallel Integer 0 number of parallel CPUs to use

NOTE

1) The uploaded image(s) must be the binarized image(s).
2) The uploaded image variable name and specified parameters' name must be same with the parameters name in the above table.
